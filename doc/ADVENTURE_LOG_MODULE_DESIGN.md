## 模块概述与定位

### 模块定位

冒险日志模块负责记录玩家在游戏中的重要事件和经历,包括战斗、任务、物品获取、等级提升等。该模块为玩家提供游戏历程的回顾功能。

### 核心职责

| 职责 | 描述 |
|------|------|
| 日志记录 | 记录游戏中的各类事件 |
| 日志查询 | 提供日志查询功能 |
| 日志管理 | 管理日志的添加和清除 |
| 数据持久化 | 实现日志数据的本地存储与加载 |

### 模块边界

**冒险日志模块**与以下模块交互:
- 战斗模块:战斗事件
- 任务模块:任务事件
- 角色模块:升级事件
- 背包模块:物品获取事件

## 接口定义

### 服务接口 IAdventureLogService

```typescript
export interface IAdventureLogService {
  getLogs(): LogEntry[];
  addLog(message: string, type?: LogType): void;
  clearLogs(): void;
  reset(): void;
}
```

### 数据类型定义

```typescript
export type LogType = 'info' | 'combat' | 'quest' | 'item' | 'level';

export interface LogEntry {
  id: string;
  timestamp: number;
  type: LogType;
  message: string;
  icon: string;
}
```

## 业务逻辑流程

### 添加日志流程

1. 调用 `addLog(message, type)` 方法
2. 生成日志条目(包含ID、时间戳、类型、消息)
3. 根据类型添加对应图标
4. 添加到日志列表
5. 保存到存储

### 清除日志流程

1. 调用 `clearLogs()` 方法
2. 清空日志列表
3. 清除存储

## 数据模型与存储设计

### IndexedDB 存储结构

| 数据库 Store | Key | 数据结构 | 说明 |
|--------------|-----|----------|------|
| adventureLog | 'adventureLog' | AdventureLogData | 冒险日志完整数据 |

### AdventureLogData 存储内容

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `id` | string | 'adventureLog' | 唯一标识 |
| `logs` | LogEntry[] | [] | 日志条目列表 |
| `updatedAt` | number | Date.now() | 最后更新时间 |

### 日志条目结构

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `id` | string | Date.now().toString() | 唯一标识 |
| `timestamp` | number | Date.now() | 创建时间戳 |
| `type` | LogType | 'info' | 日志类型 |
| `message` | string | - | 日志内容 |
| `icon` | string | 自动生成 | 类型对应的图标 |

### 日志类型图标映射

| 类型 | 图标 |
|------|------|
| `info` | 📜 |
| `combat` | ⚔️ |
| `quest` | 📋 |

## 异常处理机制

### 异常类型与处理策略

| 异常类型 | 触发条件 | 处理策略 |
|----------|----------|----------|
| 存储读取失败 | IndexedDB 解析错误 | 使用空数组初始化 |
| 存储写入失败 | IndexedDB 写入异常 | 进入重试队列，指数退避重试 3 次 |

## 性能与安全考量

### 性能优化

| 优化点 | 实现方式 | 预期效果 |
|--------|----------|----------|
| 内存缓存 | 内存存储日志列表 | 快速访问 |
| 防抖同步 | 500ms 延迟合并写入 | 减少 IO 操作 |
| 批量写入 | SyncEngine 批量处理 | 提升性能 |
| 异步加载 | Store 初始化时异步从 IndexedDB 读取 | 不阻塞主线程 |

### 数据安全

| 安全措施 | 实现方式 |
|----------|----------|
| 输入验证 | 消息长度限制 |
| 数据隔离 | 使用独立对象存储 |
| 异常捕获 | 防止程序崩溃 |
| 重试机制 | 失败时自动重试 3 次 |
| 数据校验 | 写入前验证数据结构 |
